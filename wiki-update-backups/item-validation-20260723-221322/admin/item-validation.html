Item Database & Crafting Validation

The MariaDB items table is the authoritative registry for inventory item IDs used by RDR2PF.

Source of Truth

Every crafting ingredient, crafted output, shop entry, loot reward, profession reward, farming product, and scripted inventory item must use the exact value stored in items.item.

The label column is display text only. Do not use a label in Lua or configuration files unless it is also the exact internal item ID.

SELECT item, label, `desc`
FROM items
ORDER BY item;

Required Validation Workflow

  1. Identify every ingredient and output referenced by the recipe.
  2. Query the MariaDB items table for those exact IDs.
  3. Confirm the player inventory contains the same internal ID, not merely a similar display label.
  4. Check capitalization, singular/plural differences, underscores, and spelling.
  5. Restart the affected crafting resource after approved changes.
  6. Test the recipe in game with the required quantities present.
  7. Record only confirmed working behavior in the wiki.

Useful SQL

Check specific recipe items

SELECT item, label, usable, `limit`, `desc`
FROM items
WHERE item IN ('consumable_breakfast', 'egg', 'eggs', 'pork')
ORDER BY item;

Search when the exact ID is unknown

SELECT item, label, `desc`
FROM items
WHERE item LIKE '%search%'
   OR label LIKE '%search%'
ORDER BY item;

Export the complete registry

SELECT item
FROM items
ORDER BY item;

Current Database Snapshot

The repository contains a preserved plain-text snapshot at data/item-ids.txt. It contains 997 item IDs captured from the current redm_vorp.items table.

The database remains authoritative. Regenerate the snapshot after adding, deleting, or renaming database items.

Common Failure Causes

  • The recipe uses a display label instead of the internal ID.
  • The recipe uses different capitalization.
  • The player owns egg while the recipe requires eggs, or the reverse.
  • The ingredient exists but the recipe expects a different quantity or metadata.
  • The output exists in the database but the crafting resource was not restarted after editing.
  • The recipe is registered in a different configuration file than expected.
