Development Journal

Testing Log

A concise record of validated systems and discoveries.

Foundation Milestone

Character creation, appearance saving, coordinate persistence, OneSync, stable ownership, horse recall, weapon purchase, weapon use, and txAdmin-profile startup have been validated.

Weapons

Knife, Bolt Action Rifle, rifle ammunition, bow, and arrows have been used successfully. Bow reload can be intermittent.

Hunting

At least one animal was skinned successfully. Bison, Whitetail Buck, and Wild Turkey variants require further testing.

Tools

pickaxe and lumberaxe are confirmed tool item IDs.

Custom Pause Menu and Save & Quit

Validated: The custom pause menu opens successfully, Resume works, Map opens correctly, and the Rockstar pause menu can be opened through the emergency fallback button.

Save verification: Save & Quit sends current coordinates to the server, performs the full VORP character save, waits for the MariaDB update to complete, confirms success, and then disconnects only the requesting player.

Persistence verification: Character location and money were preserved after reconnect. A one-time increase from $4200.00 to $4210.00 was observed, but the balance remained $4210.00 on the following reconnect and does not currently appear to be a repeating login reward.

Multiplayer design: The server resolves the active character from the player's server source and license rather than trusting client-supplied character identifiers.

Character Journal Development

The Character button is enabled, a dedicated Character Journal panel exists, and the Back navigation is wired. VORP already exposes live character information through LocalPlayer.state.Character, so the journal can be populated without performing a new database query each time it opens.

Inventory and Weapon Observations

The inventory money Drop action intermittently throws a DropItemMoney callback error. A failed inventory callback may leave the inventory or NUI state unusable until reconnect.

After restarting VORP resources, equipped weapons may temporarily disappear from the character's back. A full reconnect or reboot restored the bow and gun.

Los Cuervos Encounter

Implemented: Eighteen NPCs across a ten-member main camp, five-member surface position, and three-member bunker position.

Ped models: G_M_M_UniBanditos_01, G_M_M_UniCriminals_01, G_M_M_UniCriminals_02, G_M_M_UniDuster_01, A_M_M_Rancher_01, and G_M_M_UniMountainMen_01.

Current loadout: The live script forces all members to WEAPON_UNARMED both before and after aggro.

Coordinates: Main -5450.00 -3650.00 -15.00; Surface -5415.88 -3643.23 -22.17; Bunker -5395.77 -3666.90 -25.01.

Zombie Horde Alpha

Validated: Twenty zombies spawn near 2464.84 112.36 72.54, roam, maintain loose horde cohesion, detect players, and pursue correctly after a full server restart.
